How this works

The page is rendered by Python using a `resume` dictionary. Keys are accessed as template variables, for example:
  • {{ '{{ resume.name }}' }} — outputs the candidate's name.
  • {{ "{{ resume['contact']['email'] }}" }} — alternative key-style access for nested data.
The Python backend loads the dictionary and passes it when rendering the template (e.g. render_template('resume.html', resume=resume)).
